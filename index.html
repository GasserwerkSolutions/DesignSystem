Dark-Mode

Orthogonal zu Tones — data-mode="dark" kombiniert sich mit jedem Tone. Ohne data-mode wird prefers-color-scheme respektiert; data-mode="light" überstimmt System-Setting.

Aktivierung 
<html data-mode="dark">       <!-- manuell -->
<html data-mode="light">      <!-- manuell hell, ignoriert System -->
<html>                        <!-- Auto via prefers-color-scheme -->
Was sich ändert
  • BG Hintergründe dunkler, Text hell
  • Shadow Stärker (mehr Opacity) für Tiefe auf dunklem BG
  • Border Heller statt dunkler (gray-700 statt gray-200)
  • Tone Tone-Farben bleiben erhalten — dark-mode überschreibt nur Neutrale

Klick auf Dark oben rechts um die ganze Seite zu wechseln. Alle 29 Components adaptieren sich.

Accessibility

  • Tab-Navigation — Focus-Ring nutzt --focus-ring, per Theme gefärbt
  • prefers-reduced-motion — OS-Setting abschalten: alle Animationen stoppen global
  • prefers-contrast: more — Borders & Text dunkeln automatisch nach
  • Browser-Zoom / Font-Size — Strg + Scroll: komplette UI skaliert dank rem
  • Windows High-Contrast-Mode — Focus-Outline nutzt System-Highlight-Color
  • Logische Properties — padding-block, margin-inline → RTL-ready

Architektur

Layer-Reihenfolge per CSS @layer — Priorität ist spezifikations-deterministisch, nicht konventions-basiert: 

reset  <  tokens  <  semantic  <  themes  <  base  <  state  <  components

Regeln

  • Themes setzen nur Token-Werte — nie Component-Selektoren
  • Components lesen nur Tokens — nie theme-spezifische Farben direkt
  • Durchgesetzt via npm run lint (Contract-Lint)
  • WCAG-AA verifiziert pro Theme × Token-Paar via npm run check:contrast

Zahlen

  • 6 Tones × 2 Modes = 12 visuelle Zustände
  • 29 Components, alle Contract-basiert
  • 221 Tokens (Primitive + Semantic)
  • Export: tokens.json (W3C DTCG) + tokens.d.ts
